프로젝트 주제

다른 사람들이 올린 피드를 봅니다. 피드를 본 사람들이 습관 형성을 위해 데일리 인증을 시작합니다. 21일간 데일리 인증을 성공해 한 번에 몰아보는 타임랩스를 제작합니다. 이 타임랩스를 다른 사람들이 볼 수 있도록 공유합니다.

데일리 인증과 SNS를 결합해 위와 같은 습관 형성의 선순환을 목표로 하고 있습니다.

느낀점

이번에 새로 릴리즈된 Java 21 버전의 가장 큰 특징은 가상 스레드입니다. 이론상 Java 21 버전을 이용해 개발하는 것만으로도 좋은 성능을 낼 수 있었기 때문에 사용하게 되었습니다. 이에 따라 Spring Boot 3.2 RC-1 버전을 사용하였습니다. 현재 Spring Boot의 Tomcat이 가진 Thread per request 모델에서 기존 스레드(플랫폼 스레드)를 가상 스레드로 변경한 부분은 Apache JMeter로 부하 테스트를 진행한 결과 vhost가 300 이상일 때 1.5배의 성능 향상을 확인하였으나, 데이터베이스에 접근하는 부분이 pinning 문제가 발생하여 실제로 적용하지는 못해 아쉬웠습니다.

새로운 버전을 사용하게 되면서 Spring Security의 JWT, OAuth 2.0 인증, 인가 과정을 모두 새롭게 작성해야 했습니다. 이전 버전과 구현 방식이 조금 달랐기 때문에 공식 문서를 참고하여 원하는 기능을 구현하는 부분이 가장 어려웠습니다. 특히, 환경 설정에서 대부분이 Customizer 함수형 인터페이스로 추상화되어 있어 람다 표현식으로 보다 간결하고, 표현력이 뛰어난 코드 스타일로 재정의 할 수 있다는 점이 인상적이었습니다.

데일리 스크럼 시간 동안 습관 형성의 선순환과 관련된 API에 대한 고민을 팀원들에게 공유하였고, 같이 토론하며 최적의 해를 찾기 위해 고민하는 과정이 재밌었습니다. 특히, Database의 정렬된 데이터라는 특징을 활용한 점이 좋았다고 느꼈습니다. 최종적으로 이 API를 쿼리 N+1회, O(NlogN)에서 쿼리 1회, O(NlogN)으로 개선한 점이 제가 가장 자신있어 하는 알고리즘 분야로 성과를 낸 부분이라 더욱 좋았습니다.

마지막으로, TDD를 진행한 경험이 기억에 남습니다. Test Code를 먼저 생각하니, 메소드의 모듈화가 잘되고 있다는 느낌을 받았습니다. 이를 통해 메소드를 유지보수 하는 과정이 수월해졌고, 같이 작업하는 팀원이 제가 개발한 메소드를 잘 활용하는 모습이 좋았습니다.

저는 1개의 AWS EC2 서버에 3개의 React 프로젝트와 1개의 Spring Boot 프로젝트를 모두 성공적으로 파이프라인을 구축하는 성과를 이루어냈습니다. 이로써 다른 개발자들이 단순히 커밋을 하는 것만으로도 배포가 되어 더욱 개발에만 집중할 수 있는 환경을 제공한 것이 가장 뿌듯했습니다. 이외에도 Nginx, Docker, Jenkins 각자의 역할을 잘 이해하여 이번 프로젝트 상황에 맞춰 사용했다는 점 또한 뿌듯했습니다.

물론, 아쉬운 점도 있었습니다. 다양한 금융 결제 API들을 비교했지만, 저희가 원하는 것은 많은 은행 지원과 높은 신뢰도였습니다. 그래서 많은 은행을 지원하고, 일반적으로 유명한 기업인 Toss의 가상 결제 API를 선택하였습니다. 이 부분에서 Webhook을 통해 실시간 결제 상태를 업데이트 했지만, 과연 진짜 Toss의 Webhook일까?에 대한 고민에 Toss의 URL 요청인가?, 올바른 결제 키를 가졌는가?와 같은 고민을 더 깊게 못한 점이 아쉽습니다.

마지막으로, 제 다양한 기술 스택 경험의 장점이 드러날 수 있어서 좋았습니다. Python Tkinter와 React.js 경험을 활용해 다른 사람들이 이해하기 쉽도록 예제 코드를 제공해 팀 프로젝트 중에서 개발 예상 기간을 1주 단축시키는 등 실질적인 도움이 되었던 점이 가장 기억에 남습니다.

주요 기능
  • 자기소개서 작성

  • 글자 수 세기
기여한 내용
  • JWT 인증, 인가 시스템 개발 (Spring Security Framework 기반)

    • 수정, 삭제 연산이 O(N)에서 O(1)로 개선
  • 자기소개서 자동 저장, 개인 프로필 관리 개발 (React.js 기반)
