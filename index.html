Timing
프로젝트 소개

다른 사람들이 올린 피드를 봅니다. 피드를 본 사람들이 습관 형성을 위해 데일리 인증을 시작합니다. 21일간 데일리 인증을 성공해 한 번에 몰아보는 타임랩스를 제작합니다. 이 타임랩스를 다른 사람들이 볼 수 있도록 공유합니다.

데일리 인증과 SNS를 결합해 위와 같은 습관 형성의 선순환을 목표로 하고 있습니다.

주요 기능
  • 데일리 인증하기
  • 타임랩스 제작하기
  • 타임랩스 공유하기
  • 다른 사람들의 타임랩스(피드) 보기
기여한 내용
  • 총 456 커밋 중, 161 커밋 기여 (35.3%)
  • JWT, OAuth 2.0 인증, 인가 시스템 개발 (Spring Security Framework 기반)
  • SNS 피드, 댓글, 좋아요 RESTful API 개발
  • 피드 Streaming API 개발 (AWS S3 stream 방식)
  • 피드 조회 API 성능 개선
    • Schema에 Tree, Disjoint Set 자료구조 적용
    • 쿼리 N+1회, O(NlogN)에서 쿼리 1회, O(NlogN)으로 개선
  • MariaDB Schema 정의
    • SNS 피드에 대한 비정규화
  • 더미 데이터 삽입 페이지 개발 (React.js 기반)
    • 더미 데이터 하나를 삽입하는 과정이 복잡하여, 한 번에 처리하기 위해 개발
    • 1개의 데이터 삽입 시간이 약 30분에서 약 2분으로 단축
느낀점

이번에 새로 릴리즈된 Java 21 버전의 가장 큰 특징은 가상 스레드입니다. 이론상 Java 21 버전을 이용해 개발하는 것만으로도 좋은 성능을 낼 수 있었기 때문에 사용하게 되었습니다. 이에 따라 Spring Boot 3.2 RC-1 버전을 사용하였습니다. 현재 Spring Boot의 Tomcat이 가진 Thread per request 모델에서 기존 스레드(플랫폼 스레드)를 가상 스레드로 변경한 부분은 Apache JMeter로 부하 테스트를 진행한 결과 1.5배의 성능 향상을 확인하였습니다.

새로운 버전을 사용하게 되면서 Spring Security의 JWT, OAuth 2.0 인증, 인가 과정을 모두 새롭게 작성해야 했습니다. 이전 버전과 구현 방식이 조금 달랐기 때문에 공식 문서를 참고하여 원하는 기능을 구현하는 부분이 가장 어려웠습니다. 특히, 환경 설정에서 대부분이 Customizer 함수형 인터페이스로 추상화되어 있어 람다 표현식으로 보다 간결하고, 표현력이 뛰어난 코드 스타일로 재정의 할 수 있다는 점이 인상적이었습니다.

선한 영향력을 어떻게 부각시킬 수 있을까?에 대해 깊게 고민하였습니다. 저는 데일리 스크럼 시간 동안 이 문제를 팀원들에게 공유하였고, 같이 토론하며 최적의 해를 찾기 위해 고민하는 과정이 재밌었습니다. 특히, Database의 정렬된 데이터라는 특징을 활용한 점이 좋았다고 느꼈습니다. 최종적으로 이 API를 쿼리 N+1회, O(NlogN)에서 쿼리 1회, O(NlogN)으로 개선한 점이 제가 가장 자신있어 하는 알고리즘 분야로 성과를 낸 부분이라 더욱 좋았습니다.

마지막으로, TDD를 진행한 경험이 가장 좋았습니다. Test Code를 먼저 생각하니, 메소드의 모듈화가 잘되고 있다는 느낌을 받았습니다. 이를 통해 메소드를 유지보수 하는 과정이 수월해졌고, 같이 작업하는 팀원이 제가 개발한 메소드를 잘 활용하는 모습이 좋았습니다.
